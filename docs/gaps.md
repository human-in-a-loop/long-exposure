# Deferred Gaps

| Gap | Impact | Justification | Status |
|---|---:|---|---|
| Compatibility names still say `ClaudeCliError`, `ClaudeRateLimitError`, and `call_claude`. | Medium | Renaming would touch many call sites and docs. Provider behavior can be generalized underneath without disrupting the architecture. | Deferred |
| Codex MCP config flag support. | Medium | Local `codex exec --help` did not expose a direct `--mcp-config` flag. Adding unverified flags would make Codex runs fragile. | Deferred |
| Codex reasoning effort flag mapping. | Low | Local `codex exec --help` does not document a stable effort flag. Effort remains in prompts; model selection still works. | Deferred |
| Codex temp-directory trust behavior. | Low | Codex rejected an untrusted temp workspace during smoke testing. Real long-exposure runs already use an explicit `working_directory`; operators should trust that directory through Codex. | Deferred |
| README and all operator docs fully rewritten as provider-neutral. | Low | Critical setup docs will be updated after implementation; deep wording cleanup is lower value than behavior verification. | Deferred |
| Generic local connector does not have provider-native conversation sessions. | Low | The local connector is an unsupported OpenAI-compatible extension point. Long-exposure can maintain per-agent JSONL transcript logs, bounded recent-log injection, and local compaction into `sessions.db`, but backend-native session semantics depend on the operator's server. | Known limitation |
| Local provider has no native subagent/team equivalent. | Medium | Implementing a custom local subagent scheduler would duplicate long-exposure fan-out and add fragility. Whole-cycle fan-out remains available; local intra-turn subagents are deferred. | Deferred |
| Gemini free-tier native subagents are not load-bearing. | Medium | Gemini native subagents are explicitly disabled in long-exposure. Whole-cycle fan-out remains available and was live-tested with concurrent Gemini sessions. | Known limitation |
| Gemini MCP session-search wiring is project-settings based. | Medium | Long-exposure writes `.gemini/settings.json` with the `sessions` MCP server and current tool allowlist. Gemini's `mcp list` command did not reliably display project settings in a smoke test, but normal Gemini tool execution under the generated settings worked. | Partially addressed |
| Gemini tool allowlist translation is coarse. | Medium | Gemini receives `tools.core`, `tools.allowed`, and `--allowed-tools` mapped from long-exposure `allowed_tools`; command-prefix Bash mappings are supported. Exact Claude-style path scoping still relies on `cwd`, prompt guidance, and external sandboxing. | Partially addressed |
| Gemini multi-account pooling is disabled. | Medium | Long-exposure intentionally ignores `GEMINI_ACCOUNT_POOL` and `GEMINI_HOMES` for Gemini. Fan-out works through concurrent sessions on the active Gemini CLI account; account rotation needs separate OAuth validation before it should be supported. | Known limitation |
| Self-hosted open-model deployment on Google Cloud is not cost-competitive for this project. | High | Gemini Flash on the free-tier path is the cheapest practical Google-hosted option. Self-hosted open models remain possible through the generic local connector, but are not a native long-exposure integration path. | Known limitation |
| Generic local connector has no executable tool bridge. | High | OpenAI-compatible chat completions can read tool guidance, but cannot call long-exposure tools without a separate tool-calling loop and sandbox policy. | Known limitation |
| Report formatter does not rewrite fragmented pseudo-tables into Markdown tables. | Low | Automatically repairing PDF-extracted or hand-wrapped pseudo-tables risks changing data. The renderer wraps verbatim blocks reliably; semantic table reconstruction is a human/editorial task when needed. | Deferred |
| Report formatter does not auto-generate explicit heading IDs for exotic headings. | Low | Normal heading spacing now prevents the common Pandoc TOC failure. Auto-slugging every heading could create noisy diffs and accidental duplicate IDs; explicit IDs remain available for rare collisions. | Deferred |
| Report render validation does not automatically raster-preview representative PDF pages in production runs. | Low | The core pipeline now uses deterministic settings and render failures are surfaced. Automatic `pdftoppm` previews add optional system-tool coupling and file churn; manual preview remains a troubleshooting step. | Deferred |
| Manager drift detection is mostly agentic rather than a deterministic topic classifier. | Low | The poller passes recent reports and directive context to the manager agent when action is already needed, but it does not maintain a bespoke keyword ontology per campaign. A generic deterministic drift classifier would be noisy without campaign-specific labels. | Deferred |
| Manager agent sessions are not persisted back into exploration_state.json. | Low | Avoiding state writes from the cron sidecar prevents races with the main conductor. The manager receives deterministic snapshots each poll, so persistent conversational memory is a convenience rather than required state. | Deferred |
| Main cycle does not alert on missed cron manager polls by schedule. | Low | The manager writes an assessment log every poll, but the conductor does not know the operator's cron cadence and therefore cannot distinguish intentional disablement from missed polls without extra configuration. | Deferred |
| Manager no-op polls still append `_manager/no-action` ledger events. | Low | This preserves a structured audit trail for every cron poll, but very frequent cron schedules can add ledger noise over multi-week runs. Assessment logs carry the detailed record; ledger compaction or no-op sampling can be added later if real runs show bloat. | Deferred |
| Setup does not run an LLM agent to repair environment failures automatically. | Low | First-run setup happens before provider credentials are reliably configured, so deterministic installer + JSON doctor output is the robust MVP. The JSON report is designed to be handed to an agent or issue tracker for debugging. | Deferred |
| Tectonic fallback installers beyond native package managers are not bundled. | Low | `long-exposure-setup` supports apt/dnf/yum/pacman/zypper/brew/winget and reports exact missing tools if a platform repository lacks `tectonic`. Pulling installers from cargo or upstream release URLs would add network/toolchain variability; revisit only if package-manager coverage proves insufficient. | Deferred |
| CLI adapter installer is template-based, not version-introspective. | Medium | `long-exposure cli-install` writes deterministic Claude/Codex/Gemini routing files with overwrite safeguards, but it does not inspect every installed CLI version to prove that each hook format is auto-loaded. The files route to the robust launcher and are safe; deeper version-specific adapter discovery can be added once real CLI version drift is observed. | Deferred |
| `long-exposure launch --manager` does not force an immediate manager poll at process start. | Low | On a fresh launch the state file may not exist until the conductor initializes. The embedded manager loop waits for its configured interval and cron/headless polling still works. Forcing an early poll would mostly produce no-state noise. | Deferred |
| Launcher tailing is intentionally minimal. | Low | Foreground `long-exposure launch` streams conductor output directly and `long-exposure status` surfaces the latest manager notice. A richer multiplexed tail of run logs plus manager JSONL could improve ergonomics, but it is not necessary for reliability and would add more terminal-state handling. | Deferred |
| Live Claude/Codex/Gemini smoke tests are still opt-in/manual. | Medium | The deterministic suite now covers provider command construction, Codex/Gemini parsing, one-cycle loop behavior, resume, and fan-out without credentials. Live smoke tests require user accounts and can be flaky under rate limits, so they should remain explicitly gated rather than part of normal tests. | Deferred |
| Telemetry does not parse provider-native inner tool transcripts. | Medium | The passive telemetry layer records Python-visible events, provider usage envelopes, report outcomes, fan-out outcomes, manager verdicts, and deterministic rollups. Parsing Claude/Codex/Gemini internal tool transcripts is provider/version-specific and should be added only after stable transcript fixtures exist. | Deferred |
| Telemetry lessons are deterministic scaffolds, not agent-authored analyses. | Low | `telemetry summarize` writes a lessons shell with evidence prompts. Running an analyst agent over telemetry is useful later, but keeping it out of the live path preserves passive behavior and avoids hallucinated causal claims during runs. | Deferred |
| Telemetry export is local-only. | Low | The MVP intentionally avoids network upload and sharing flows. A future redacted export bundle can be added after operators validate which fields are useful and safe to share. | Deferred |
| Telemetry path redaction defaults to off for local analysis. | Low | Prompt/response/tool/env fields are centrally omitted by default and account path keys are hashed, but normal artifact paths remain visible unless `redact_paths: true` is set. This preserves local debugging usefulness; a future share/export command should force path redaction. | Deferred |
| No automatic migration for old root-level final-stage scratch files. | Low | New runs route scratch to `reports/final/` and `audits/final/`, and `org_check` notes legacy root artifacts without failing. Moving historical files automatically could surprise operators; a one-shot migration helper is only worth adding if real workspaces accumulate confusing legacy clutter. | Deferred |
| Delta mode does not resume inside a partially completed final-auditor/final-reporter stage chain. | Medium | The run writes `run_mode.json` and reuses deterministic scratch paths, but a killed delta pass restarts the staged pass instead of continuing at the next missing stage. Restarting is safe and simpler; stage-level resume can be added later if long delta passes make the wasted work material. | Deferred |
| Legacy-size delta detection cannot compute a precise new-artifact boundary. | Low | Explicit `*.committed` markers are used for new successful outputs and provide a timestamp boundary. Existing pre-marker workspaces fall back to the old size heuristic for compatibility, which may over-budget one continuation but avoids silently treating a prior final report as absent. | Deferred |
